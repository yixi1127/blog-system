# 博客文章管理系统 - 使用说明

## 项目概述

这是一个基于 Vue3 + TypeScript + Ant Design Vue 开发的博客文章管理系统前端项目。系统实现了真实的数据存储功能，使用浏览器 localStorage 进行数据持久化。

## 已实现功能

### ✅ 文章管理
- **文章列表**：展示所有文章，支持搜索、筛选（按分类、状态）、分页
- **新建文章**：创建新文章，支持标题、分类、标签、内容、摘要、发布状态
- **编辑文章**：修改已有文章的所有信息
- **查看文章**：查看文章详情，包括统计信息（阅读量、点赞数、评论数）
- **删除文章**：删除不需要的文章
- **文章预览**：编辑时可预览文章效果

### ✅ 分类管理
- **分类列表**：展示所有分类及文章数量
- **新建分类**：创建新的文章分类
- **编辑分类**：修改分类名称、描述、排序
- **删除分类**：删除不需要的分类
- **自动统计**：自动统计每个分类下的文章数量

### ✅ 数据持久化
- 所有数据保存在浏览器 localStorage 中
- 刷新页面数据不丢失
- 真实的增删改查操作，不是虚假数据

## 技术架构

### 前端技术栈
- **Vue 3**：使用 Composition API
- **TypeScript**：类型安全
- **Ant Design Vue**：UI 组件库
- **Vue Router**：路由管理

### 数据存储方案
- **存储位置**：`src/later/` 文件夹
- **storage.ts**：数据存储服务层，负责与 localStorage 交互
- **api.ts**：API 服务层，模拟后端 API 调用
- **存储介质**：浏览器 localStorage

### 数据存储位置
所有数据存储在浏览器的 localStorage 中，键名如下：
- `blog_articles`：文章数据
- `blog_categories`：分类数据
- `blog_user`：用户信息
- `blog_article_id`：文章 ID 计数器
- `blog_category_id`：分类 ID 计数器

### 清除数据
如需清除所有数据，可以：
1. 打开浏览器开发者工具（F12）
2. 进入 Application/应用 标签
3. 找到 Local Storage
4. 删除以 `blog_` 开头的所有键
5. 刷新页面，系统会重新初始化默认数据

## 使用流程示例

### 创建一篇新文章
1. 点击导航栏的"文章列表"
2. 点击"新建文章"按钮
3. 填写文章标题，例如："React Hooks 使用指南"
4. 选择分类，例如："技术文章"
5. 添加标签，例如：React、Hooks、前端
6. 编写文章内容
7. 填写摘要（可选）
8. 选择"立即发布"
9. 点击"发布文章"按钮
10. 系统自动跳转到文章列表，可以看到新创建的文章

### 管理分类
1. 点击导航栏的"分类管理"
2. 点击"新建分类"按钮
3. 填写分类名称，例如："项目实战"
4. 填写分类描述，例如："实际项目开发经验分享"
5. 设置排序值，例如：4
6. 点击"确定"
7. 新分类创建成功，可以在创建文章时使用

### 编辑文章
1. 在文章列表中找到要编辑的文章
2. 点击"编辑"按钮
3. 修改文章内容
4. 点击"保存修改"
5. 修改成功，自动跳转到文章列表

## 注意事项

### ⚠️ 数据安全
- 数据仅存储在本地浏览器
- 清除浏览器缓存会导致数据丢失
- 不同浏览器/设备之间数据不共享
- 建议定期备份重要内容

### ⚠️ 存储限制
- localStorage 有大小限制（通常 5-10MB）
- 不建议存储大量文章或大型图片
- 单篇文章内容建议不超过 10000 字符

### ⚠️ 浏览器兼容性
- 需要支持 localStorage 的现代浏览器
- 推荐使用 Chrome、Firefox、Edge 最新版本

## 常见问题

### Q: 数据会丢失吗？
A: 只要不清除浏览器缓存，数据会一直保存。但建议重要内容做好备份。

### Q: 可以在不同设备上访问吗？
A: 不可以。数据仅存储在当前浏览器的 localStorage 中，不同设备无法共享。如需多设备访问，需要对接真实后端 API。

### Q: 如何备份数据？
A: 可以打开浏览器开发者工具，在 Application → Local Storage 中复制数据，保存到文本文件。

### Q: 如何恢复数据？
A: 将备份的数据粘贴回 Local Storage 对应的键中，刷新页面即可。

### Q: 可以存储多少文章？
A: 取决于浏览器的 localStorage 限制（通常 5-10MB）。建议不超过 100 篇文章。
